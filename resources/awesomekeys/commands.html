Commands

General commands

/k new lock <lock name>

Define a new lock. The name has to be kept safe to create keys.

/k new key <lock name>

Create a new key for the specified lock.

/k add lock <lock name>

Add a lock to a targeted lockable block. The lock has to be defined first.

/k add code

Add a code to a targeted lockable block, instead of classic lock. No key needed to access it then, just a code.

/k remove

Remove a lock from the targeted block. Breaking the block also removes the lock.

/k greeting none|<color code> <greeting message>

Add a greeting message to a block you locked. It will be displayed each time a player is accessing it. Only works for doors and fence gates.
Color code is a Minecraft color code character among the following : 0123456789abcdef
Just type "none" to remove the message.

/k <editkey line> <description>

Add a description to the key. The description length cannot exceed 50 characters for each line, and there can be up to 4 lines of description.

OP and staff commands

/k master

Create a master key, which can unlock any block.

/k logs <filters> (> <file>)

Print logs to a file or to the chat. Please refer to the logs section.

/k purge unlinked|all|player (<player>)

Purge some files that could have become useless.
- unlinked : if you deleted locked blocks with a plugin (eg. WorlEdit), AwesomeKeys has no way to know it ; it may result as lock files linked to nothing ; you can purge them easily if you think there are some
- all : purge all AwesomeKeys, except configuration and languagew
- player : purge all files related to the specified player ; if a player is banned, the plugin keeps all files related to him

/akverifylang <lang> (<page>)

You can see easily how the messages are displayed with the language selected. The language has to be installed.
