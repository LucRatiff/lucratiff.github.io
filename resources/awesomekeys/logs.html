Logs

AwesomeKeys registers logs whenever a locked block is accessed. If you want to control what kind of logs are registered, please refer to the configuration section.

Print logs

You can print the logs to a file or to the chat via a private message. In order to do that, you need the permissions "awesomekeys.logs.print" and "awesomekeys.logs.read".

The command is as follows :

/k logs <filters> (> <file>)

The file, if spectified, will be created in the data folder (plugins/AwesomeKeys)

Filters

You have to add at least one filter, and then add others filters right after, all seperated by spaces.

target

Print logs of the targeted block, the one you are selecting with the pointer.

block <world> <x> <y> <z>

The block which is in those exact coordinates.

group [chunk]|[world (<name>)]|[all]

There are three possible groups :
- chunk, which is the chunk of your current position ;
- world, the world of your position, or the world spectified in <name> ;
- all, all worlds

player <name|uuid>

A player, by its name or its uuid.

from <dd/mm/yy,hh(:mm(:ss))>|beginning to <date>|now

The time filter. For example, if you want logs from the beginning to 01/01/2022 at 9:30 pm of a targeted block, and print it to logs.txt :

/k logs target from beginning to 01/01/2022,21:30 > logs.txt
