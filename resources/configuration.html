Configuaration

AwesomeKeys is deeply customizable through its configuration file.

The configuration file

The file is named "config.json" and is placed in the plugin data folder "plugins/AwesomeKeys". If you never edited a json file, you can refer to this page (optionnal) : Wikipedia:JavaScript_Object_Notation

If you have a problem with the configuration file, you can delete it, and it will be generated from default.

The fields

keys

		"keys": [
			{
				"item": 280,
				"durability": [ ]
			}
		],

Defines the items that could be used as keys

item: You can use either id numeric value or 1.13+ flattening text value (use quotes)

durability: None, one or mutliple durability/data values, separated by a comma

If you want to add multiple items, copy and paste this block as follows, and add a comma in between :

language

"language": "en",

The language wich will be loaded by the plugin. Please refer to Language

alias-command-key

"alias-command-key": "k",

Alias for command /key, default is /k

enable-logs

"enable-logs": true,

If logs are enabled, they track all interactions with locked blocks : creation, removal, access, destruction, etc. You can then consult them and filter them as mentionned in

buffer-duration

"buffer-duration": 20,

The maximum amount of time in seconds some objects are buffering. Don't change it except if your server has too few memory

deny-crafting-key-exact-items

"deny-crafting-key-exact-items": false,

By default, players are allowed to craft the exact item (excluding tags). In any situation, it won't be a key, because of tags.

deny-crafting-key-item-types

"deny-crafting-key-item-types": false,

Deny crafting items that have the same type, regardless the durability/data value

deny-receiving-key-exact-items

"deny-receiving-key-exact-items": false,

receiving includes mob loot and NPC exchange.

Important: Don't set it to true if you have custom NPCs that give keys to players

deny-receiving-key-item-types

"deny-receiving-key-item-types": false,

Same thing, regarless the durability/data value

deny-renaming-key-items

"deny-renaming-key-items": true,

By default, players cannot rename their keys ; they can only edit the description. Renaming a key don't change its functionning

per-player-containers-limitation

"per-player-containers-limitation": 0,

The maximum amount of containers a player can lock. 0 is limitless

per-player-doors-limitation

"per-player-doors-limitation": 0,

The maximum amount of doors and fence gates a player can lock. 0 is limitless

per-player-locks-limitation

"per-player-locks-limitation": 500,

The maximum amount of locks a player can define. 500 is enough to avoid disk space spam

enable-*

		"enable-wooden-doors": true,
		"enable-iron-doors": true,
		"enable-fence-gates": true,
		"enable-chests": true,
		"enable-trapped-chests": true,
		"enable-shulkers": true,
		"enable-other-containers": false,

Enable or disable each kind of block that can be locked. By default, only the containers that are chest (normal and trapped, simple and double) and shulkerbox can have a lock.

enable-particles

"enable-particles": true,

Modify it only if your server has severe performance issues

bypass-protection

"bypass-protection": true

By default, players can access a locked block with the propper key, even if the access is restrected by a protection plugin
