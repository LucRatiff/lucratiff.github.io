Locks

There are two types of locks : classic locks and codes. Classic locks need a key to be opened, and codes are 9 digit codes composed from a special 3x3 inventory. You can lock any door, fence gate, chests and shulkers. Other containers are not lockable by default, but they can be in the configuration file.

Define a new lock

In order to add a lock to a block, you have to define first a lock. Consider it as making a model of the lock before forging locks based on the plan. That lock will have a name and a uuid (unique identifier). Keep the name of your lock in order to make keys.

/k new lock <name>

To create new keys, please refer to the keys section

Add a lock to a block

By default, there is no limit in the amount of blocks that you can lock. To lock a new bloc, select it with your cursor and then type the command for a classic lock :

/k add lock <name>

By default, anyone who has the propper key can access the locked block, even if without the lock, the block couldn't be accessed due to a protection plugin. It can be changed in the config file.

You can also add a code lock :

/k add code

You'll be invited to enter a code on a 3x3 inventory. You can enter up to 12 digits by default. Keep safe your code.

Remove a lock

To remove a lock from a block you locked, select it with your cursor and type this command :

/k remove

The lock has to be yours. Note that if the block is broken, the lock is also removed, even if you place the same block at the same location.

Add a greeting message

You can add a colorful greeting message to all your locks ! It works only for doors and fence gates (containers inventories would hide the message). Type this command :

/k greeting <color code> <message>

The color code has to be one the following : 0123456789abcdef ; wich represent Minecraft color code.

Note that if you use a resource pack with smaller fonts, some players might lose the end of the message if you entered too many characters (like signs).

Shulkerboxes

If you add a lock to a shulker, it won't be removed on block break. It keeps the lock uuid in its lore.

Protection management

The protection from the destruction is not handled by this plugin. In order to do that, you have to use a protection plugin like WorldGuarg or GriefPrevention.

You can choose in the config file wether if the protection is consulted or not when accessing a locked block ("bypassprotection" field). For example, if "bypassprotection" is set to false, AwesomeKeys won't consider the protection. If true, the player that is trying to access the block with the propper key has to have the permission to access it in a normal way.
